# **Milestone 1**

---
# System Architecture

This project follows a modular multi-robot ROS2 architecture. Each robot (Hunter, Bunny, Wolf) runs its own control nodes, while leveraging shared navigation and localization infrastructure.

---

## Shared / Standard ROS Components

### Localization
- **AMCL** (Adaptive Monte Carlo Localization) on a known map  
  _Alternative:_ SLAM Toolbox (if online mapping is required)

### Navigation
- **Nav2** stack for:
  - Global planning
  - Local control
  - Path execution
  - Costmaps

### TF Tree
- `map → odom → base_link`
- All inter-robot reasoning is performed in the **map frame**

---

# Role-Specific Nodes

## 🐰 Bunny (Reactive Prey)

### `bunny_wander_node`

**Subscribes to:**
- `/scan` (LiDAR obstacle data)
- `/odom`

**Publishes:**
- `/cmd_vel`

**Behavior Logic:**
- Move forward at constant velocity
- If obstacle detected within threshold:
  - Stop
  - Rotate by random angle
  - Resume forward motion
- No global reasoning or planning

This provides stochastic prey behavior for the Hunter to predict and pursue.

---

## 🐺 Wolf (Ambush Predator)

### `wolf_ambush_node`

**Subscribes to:**
- `/scan`
- `/odom`
- Optional: `/tf` (for map awareness)

**Publishes:**
- `/cmd_vel`

**Behavior Logic:**
- Prefer corner regions / occluded spaces
- Use obstacle geometry to hide from line-of-sight
- Opportunistically move toward Hunter if within detection range
- Remains mostly local and reactive (no global pursuit planning)

The Wolf introduces adversarial risk into the environment.

### 3️⃣ `hunter_planner_node` (Custom Algorithm)

This node implements our primary algorithmic contribution:

# Risk-Aware Pursuit Planning

Instead of directly chasing the Bunny, the Hunter performs **goal selection under risk constraints**.

---

## Core Idea

We construct a **danger field** around the Wolf and plan toward the Bunny while minimizing exposure to this danger.

---

## Wolf Danger Cost Field

For any position \( x \) in the map:

\[
C_{wolf}(x) =
\begin{cases}
\text{Very Large Penalty}, & d(x, wolf) \le R_{kill} \\
\exp(-\alpha (d(x, wolf) - R_{kill})), & d(x, wolf) > R_{kill}
\end{cases}
\]

Where:
- \( d(x, wolf) \) = Euclidean distance to Wolf
- \( R_{kill} \) = kill radius
- \( \alpha \) = decay rate

This creates:
- Extremely high cost inside kill radius
- Gradually decaying danger outside it

---

## Objective Function

Where:
- `PathCost_to_Bunny` comes from Nav2 global planning
- `Integrated_Wolf_Danger` is accumulated danger along the candidate path
- `λ` controls risk aversion

---

## Output

The planner produces:
- A **safe intermediate goal pose** for Nav2  
  OR
- Direct velocity override (if implementing custom control)

The result:
- Hunter approaches Bunny
- Hunter naturally skirts Wolf’s danger region
- No explicit hard-coded avoidance rules required

---

### 4️⃣ `hunter_safety_watchdog_node`

**Monitors:**
- Distance to Wolf
- Planner heartbeat
- Sensor data availability

**Emergency Actions (if triggered):**
- Publish zero velocity
- Cancel Nav2 goal
- Mark run as failed

# Evaluation Plan

We will evaluate over **N = 20 randomized trials**.

## Metrics

- Hunter win rate (% successful captures)
- Average time-to-capture
- Death rate (% entering kill radius)
- Minimum Hunter–Wolf distance during run
- Path length and navigation smoothness

---

# Safety Protocol

Even in simulation, strict safety logic is enforced:

- **Kill radius enforcement:**  
  If \( d(H, W) \le R_{kill} \):
  - Stop immediately
  - Cancel navigation
  - End trial

- **Command timeout:**  
  If planner does not update for > 0.5s → stop robot

- **Sensor heartbeat monitoring**

- **Velocity limits:**  
  - Linear ≤ 0.3 m/s  
  - Angular ≤ 1.0 rad/s
